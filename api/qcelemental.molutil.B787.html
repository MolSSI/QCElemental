Use Kabsch algorithm to find best alignment of geometry cgeom onto rgeom while sampling atom mappings restricted by runiq and cuniq.

Parameters:

  • rgeom (ndarray) ‚Äì (nat, 3) array of reference/target/unchanged geometry. Assumed [a0] for RMSD purposes.

  • cgeom (ndarray) ‚Äì (nat, 3) array of concern/changeable geometry. Assumed [a0] for RMSD purposes. Must have same nat, units, and atom content as rgeom.

  • runiq (ndarray) ‚Äì (nat,) array of str indicating which rows (atoms) in rgeom are shuffleable without changing the molecule. Generally hashes of element symbol and mass are used, but could be as simple as [‚ÄòC‚Äô, ‚ÄòH‚Äô, ‚ÄòH‚Äô, ‚ÄòD‚Äô, ‚ÄòH‚Äô] for monodeuterated methane.

  • cuniq (ndarray) ‚Äì (nat,) array of str indicating which rows (atoms) in cgeom are shuffleable. See runiq for more details. Strings and count in cuniq must match runiq. That is, sorted(cuniq) == sorted(runiq).

  • do_plot (bool) ‚Äì Pops up a mpl plot showing before, after, and ref geometries.

  • verbose (int) ‚Äì Quantity of printing. 0 to silence.

  • atoms_map (bool) ‚Äì Whether atom1 of rgeom already corresponds to atom1 of cgeom and so on. If True, no resorting will be run, parameters runiq and cuniq may be passed as None, and much time will be saved.

  • run_resorting (bool) ‚Äì Run the resorting machinery even if unnecessary because atoms_map=True.

  • run_to_completion (bool) ‚Äì Run reorderings to completion (past RMSD = 0) even if unnecessary because mols_align=True. Used to test worst-case timings.

  • algorithm (str) ‚Äì {‚Äòhungarian_uno‚Äô, ‚Äòpermutative‚Äô} When atoms_map=False, screening algorithm for plausible atom mappings. permutative suitable only for small systems.

  • uno_cutoff (float) ‚Äì TODO

  • run_mirror (bool) ‚Äì Run alternate geometries potentially allowing best match to rgeom from mirror image of cgeom. Only run if system confirmed to be nonsuperimposable upon mirror reflection.

Returns:

First item is RMSD [A] between rgeom and the optimally aligned geometry computed. Second item is a AlignmentMill with fields (shift, rotation, atommap, mirror) that prescribe the transformation from cgeom and the optimally aligned geometry.
